# Quant Career Desk

面向上海商品期货量化研究员的个人职业工作台。第一阶段根据本人已确认信息建立职业定位、目标岗位原型和能力证据框架：

- 上海远澜私募基金量化研究经验 6 年
- 核心工作为商品期货日频截面策略开发
- 熟练使用 Python 和大模型
- 2024 年 3 月至今，策略由公司 PM 配置约 10 亿元资金运行
- 费后实盘年化收益约 30%、Sharpe 约 2、最大回撤约 10%
- 约 10 天低频换手，容量估计在 50 亿元以上

页面把个人贡献、公司账户和最终决策权分开：本人负责公开数据信号、模型预测和组合优化；交易下单由其他同事完成，资金配置与最终风险决策由公司 PM 负责。职业目标是继续亲自研究和写策略的 IC 型策略负责人或研究型 PM，不转向纯团队管理岗。

岗位筛选固定为上海，管理规模至少 100 亿元；机构以私募基金为主，公募基金和证券公司等为辅。薪酬采用两条独立硬条件：固定税前保证收入至少 100 万元，且绩效良好年份的工资现金税后超过 80 万元。按当前可核验的上海参数、7% 公积金和无专项附加扣除估算，100 万元固定收入现金到手约 70.6 万元；全年一次性奖金单独计税时，奖金约需 10.5 万元，并入综合所得时按至少 16 万元保守筛选。页面估算不替代单位薪酬测算或专业税务意见。

## 当前范围

- `职业画像`：默认加载，展示实盘成绩单、职责边界、职业定位和研究证据链。
- `目标方向`：按需加载 `data/directions.json`，把个人证据和已核验岗位样本归纳成六类职业方向。
- `能力地图`：按需加载 `data/skills.json`，包含 6 项 P0 核心能力和 3 项 P1 差异化能力。
- `薪酬换算`：按需加载 `data/compensation.json`，可切换奖金单独计税/并入综合所得情景。
- `岗位雷达`：点击后才并行载入机构、岗位、共同技能和行动清单四份精简数据；原始公开职位快照不会发布到网页。
- 明暗主题、本地路由、桌面与移动端响应式布局。

岗位雷达把当前机会分为重点待核、相邻岗位、公募/券商、边界线索和历史参照。地点、机构规模、薪酬和职级分别判定；招聘广告没有给出固定薪与奖金结构时一律显示“待核验”，不会根据机构名声或行业传闻推断达标。行动清单给出五条优先沟通路径、停止条件、薪酬核验话术和四份脱敏面试证据。四份填写模板各自存成独立 JSON，只有点击某份模板时才加载对应文件。

## 岗位数据与访问边界

- 机构规模优先使用中国证券投资基金业协会的管理人公示，职位优先使用机构官网、官方招聘门户或雇主公开职位页。
- `scripts/collect-minghong-jobs.cjs` 通过公开页面低频读取明汯三页职位列表，只保存公开职位字段，不访问登录、申请或个人数据。
- 灵均第三方招聘路径受 `robots.txt` 限制，九坤官网条款禁止 crawler/spider，因此两者只人工浏览和核验，不运行自动采集。
- `sources/snapshots/` 保存带日期的原始公开职位快照用于审计，不在同步清单中，不会进入 GitHub Pages 资源。
- `data/job-skills.json` 的 `n/N` 只描述页面列出的定向样本，不是市场总体估计。

重新采集明汯公开职位需要工作区已有 Playwright：

```bash
NODE_PATH=/mnt/Data/jiqiang/job/node_modules \
SNAPSHOT_DATE=2026-07-24 \
node scripts/collect-minghong-jobs.cjs
```

## GitHub Pages 子站

量化站以本目录为唯一编辑源，通过脚本同步到现有招聘网站的 `quant/` 子目录：

```bash
node scripts/sync-to-recruitment.cjs
```

发布地址为 `https://jiqiangzhangnyu.github.io/recruitment/quant/`。同步后需要在 recruitment 仓库单独提交并推送。

## 数据边界

`data/profile.json` 是个人事实的唯一入口。任何绩效数字都应补全时间区间、样本定义、费用口径和是否实盘；涉及原雇主策略的材料必须经过脱敏，页面不保存代码、参数、持仓或可逆推出策略的信息。
